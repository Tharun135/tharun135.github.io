# Technical Writing Work

My documentation work spans user manuals, API docs, and internal tooling guides — 
delivered through a docs-as-code workflow using MkDocs and Fluid Topics.

## Workflow
- Write in Markdown, review in VS Code
- Publish via MkDocs to Fluid Topics portal
- Follow Program Increment (PI) release cycles — docs ship in the same sprint as features

## Areas
- User manuals for software products
- API documentation
- Internal automation and tooling documentation
