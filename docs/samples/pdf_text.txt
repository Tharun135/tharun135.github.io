Manual, 01/2026, A5E51720554-AK 91.1 Introduction
1.1.1 Introduction to SIMATIC S7+ Connector
Welcome to SIMATIC S7+ Connector documentation for Industrial Edge.
The SIMATIC S7+ Connector is a south bound connector that connects data from S7-1200/1500
PLCs, PLCSim Advanced and Siemens Software Controller with Industrial Edge. Connectors are
essential to collect data from various PLCs and devices and make it available on the Industrial
Edge system. With the S7+ Connector data can be accessed via the optimized S7 protocol.
This connector is developed as Connectivity Suite Connector and therefore supports
standardized conﬁguration as well as data transfer.
The Common Conﬁgurator conﬁgures the Connector and is available free of charge in the
Industrial Edge Marketplace.
Funded by the Federal Ministry for Economic Aﬀairs and Energy based on a decision by the
German Bundestag. Your funding provider: Kicks for Edge as part of the EU funding program
IPCEI-CIS (FKZ:13IPC008)
1.1.2 Validity of the documentation
The SIMATIC S7+ Connector for Industrial Edge documentation is valid for installing the app
on an Industrial Edge Device (IE Device). It provides users with comprehensive guidance on how
to use the application.

This product does not provide an automatic deletion for the Databus or PLC credentials already

3.2.1.1 Support for Ar rays
Arrays are provided in the Common Conﬁgurator as a tag.
SIMATIC S7+ Connector supports arrays of default data types, including String, WString and DTL.
S7+ Connector provides all individual elements of an array as separate tags and also the whole
array as one common tag in a JSON  ﬁle. The user can decide which data to work with.
There is a limit on the total size of all array elements that can be stored in a PLC. For example,
the S7-1516 has a limit of 200 KB. The limit can be diﬀerent for other models and ﬁrmware
versions.
3.2.1.2 Support of Char and String
SIMATIC PLC store characters for Char and String with 8 Bits. The ASCII standard is deﬁned
only for 7 Bit. The 2nd half of the possible values is deﬁned by the used code page.
The runtime system has no information about the used code-page in the TIA Portal. That's why
the SIMATIC S7+ Connector is using ISO/IEC 8859-1 (Latin-1) code-page.
When writing Char or String values to the PLC this code-page is used as well. Any invalid
character is replaced by U+00FF (ÿ).

4.2 Check of PLC certiﬁcates
4.2.1 Introduction
Certiﬁcates ensure secure communication in the system and typically require no user
intervention. The TIAP Export includes necessary certiﬁcates, while the Common Import
Converter embeds them within the JSON  conﬁguration. The Common Conﬁgurator forwards this
ﬁle to the S7+ Connector, which validates self-signed and CA certiﬁcates.
In cases where certiﬁcate changes or deletions are needed, this documentation provides
guidance on modifying certiﬁcates, adjusting TLS settings, and resolving common certiﬁcate-
related errors.
SIMATIC S7+ Connector Version 1.1
The secure TLS connection between S7+ Connector and PLC is supported but the PLC
certiﬁcates are not checked.
SIMATIC S7+ Connector Version 1.2
The PLC certiﬁcates are veriﬁed.
If they are not valid, the conne ction is rejected.
If no certificates are configured, the check is not performed. The connection is still
established and secured as in version 1.1. The communication partners are not
authenticated.
4.2.2 Description
You typically do not need to manage the certiﬁcates manually. In the certiﬁcate management
workﬂow, various components work together to handle and verify certiﬁcates automatically:
TIA Portal Export already contains the certificates.NOTICE

5.2.1.3 Conﬁguring the acquisition mode
The S7+ Connector supports the following acquisition modes:
OnChange
Cyclic Continuous
This setting is conﬁgured at the connection level.NO TICE

48 Manual, 01/2026, A5E51720554-AK5.2.1.3.1 OnChang e
The default setting is On Change. In On Change mode, data is only transmitted when values
change. This conﬁguration is optimal for system load for all involved components such as the
network, Databus, IIH Essentials, or IIH Semantics.
The S7+ Connector operates internally on a subscription-based model. This means that the PLC
implements change-of-value logic.
5.2.1.3.2 Cy clic Continuous
Some speciﬁc use cases might require Cyclic Continuous, such as machine learning applications.
The cyclic continuous data acquisition mode sends all data upstream each cycle, even when the
data does not change. This mode impacts the performance of the IE Device. The northbound
gRPC connections to the Databus and/or IIH Essentials / IIH Semantics experience higher load.
Storage utilization for the timeseries data in IIH Essentials also increases. The communication
with the PLC is also heightened.
5.2.1.3.3 Mix ed scenar ios
When selected data points are needed in Cyclic Continuous mode, deﬁne a separate data source
with Cyclic Continuous. This way, the common data points can be eﬃciently provided in the
On Change mode in a separate data source. Meanwhile, only the required data points are
provided in Cyclic Continuous mode.
